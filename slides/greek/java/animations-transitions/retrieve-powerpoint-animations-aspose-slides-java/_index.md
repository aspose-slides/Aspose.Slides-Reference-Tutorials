## Οδηγός υλοποίησης

### Ανάκτηση και εμφάνιση εφέ κίνησης

#### Επισκόπηση
Οι παρακάτω ενότητες δείχνουν πώς να επαναλάβετε κάθε διαφάνεια, να εξάγετε τις λεπτομέρειες κίνησης και να τις εκτυπώσετε—ιδανικό για τη δημιουργία του εργαλείου ανάλυσης κινούμενων γραφικών.

#### 1. Εισαγωγή απαραίτητων κλάσεων
```java
import com.aspose.slides.IEffect;
import com.aspose.slides.ISlide;
import com.aspose.slides.Presentation;
```

#### 2. Αρχικοποίηση του αντικειμένου Presentation
```java
String presentationFileName = "YOUR_DOCUMENT_DIRECTORY/AnimationShapesExample.pptx";
Presentation pres = new Presentation(presentationFileName);
```

#### 3. Επανάληψη μέσω διαφανειών και εφέ
```java
try {
    for (ISlide slide : pres.getSlides()) {
        IEffect[] effects = slide.getTimeline().getMainSequence();

        for (IEffect effect : effects) {
            String effectType = effect.getType();
            int targetShapeId = effect.getTargetShape().getUniqueId();
            int slideNumber = slide.getSlideNumber();

            System.out.println(effectType + " animation effect is set to shape#" +
                    targetShapeId + " on slide#" + slideNumber);
        }
    }
} finally {
    pres.dispose(); // Always dispose of the Presentation object to free resources
}
```

**Επεξήγηση**
- `getSlides()`: Ανακτά όλες τις διαφάνειες.  
- `getTimeline().getMainSequence()`: Επιστρέφει τη κύρια ακολουθία κίνησης για μια διαφάνεια.  
- `getType()` και `getTargetShape()`: Παρέχουν το όνομα του εφέ και το σχήμα που κινείται.  

#### Συμβουλές αντιμετώπισης προβλημάτων
- Επαληθεύστε ότι η διαδρομή του αρχείου είναι σωστή και το αρχείο είναι προσβάσιμο.  
- Βεβαιωθείτε ότι η έκδοση του Aspose.Slides ταιριάζει με το JDK σας (χρησιμοποιήστε τον ταξινομητή `jdk16`).

Κατά την επεξεργασία τεράστιων παρουσιάσεων:

## Συνηθισμένα λάθη & λύσεις
- **Κενό (null) σχήμα στόχου:** Ορισμένα εφέ μπορεί να μην είναι συνδεδεμένα με σχήμα (π.χ. μεταβάσεις διαφανειών). Προσθέστε έλεγχο `null` όταν καλείτε `getTargetShape()`.  
- **Διαρροές μνήμης:** Η παράλειψη του `pres.dispose()` μπορεί να προκαλέσει γρήγορη αύξηση της εγγενούς μνήμης.
