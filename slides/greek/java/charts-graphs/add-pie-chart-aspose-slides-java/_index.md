Το πακέτο Maven (`aspose-slides`) απλοποιεί τη διαχείριση εξαρτήσεων, επιτρέποντάς σας να εστιάσετε στην κατασκευή και προσαρμογή διαφανειών—όπως η προσθήκη ενός γραφήματος πίτας—χωρίς να ασχοληθείτε με τη χαμηλού επιπέδου διαχείριση αρχείων.

## Ρύθμιση του Aspose.Slides για Java
Πρώτα, συμπεριλάβετε το Aspose.Slides στο έργο σας μέσω Maven ή Gradle.

**Maven:**  
```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-slides</artifactId>
    <version>25.4</version>
</dependency>
```
```xml
<dependency>
    <groupId>com.aspose</groupId>
    <artifactId>aspose-slides</artifactId>
    <version>25.4</version>
    <classifier>jdk16</classifier>
</dependency>
```

**Gradle:**  
```groovy
implementation 'com.aspose:aspose-slides:25.4'
```
```gradle
implementation group: 'com.aspose', name: 'aspose-slides', version: '25.4', classifier: 'jdk16'
```

## Οδηγός Υλοποίησης
Παρακάτω χωρίζουμε τη λύση σε δύο δυνατότητες: την προσθήκη γραφήματος πίτας και την πρόσβαση στο βιβλίο εργασίας δεδομένων του.

### Δυνατότητα 1: Δημιουργία Παρουσίασης και Προσθήκη Γραφήματος
#### Επισκόπηση
Αυτό το τμήμα δείχνει πώς να δημιουργήσετε μια νέα παρουσίαση και **add a pie chart** στην πρώτη διαφάνεια.

#### Πώς να δημιουργήσετε pie chart aspose;
Φορτώστε την κλάση `Presentation`, προσθέστε ένα γράφημα τύπου `ChartType.Pie` και αποθηκεύστε το αρχείο. Η ολόκληρη λειτουργία απαιτεί μόνο τρεις κλήσεις API και εκτελείται σε λιγότερο από ένα δευτερόλεπτο για ένα τυπικό σετ 10 διαφανειών, καθιστώντας το ιδανικό για αυτοματοποιημένη δημιουργία αναφορών.

#### Βήμα‑βήμα

**Βήμα 1: Αρχικοποίηση Νέου Αντικειμένου Παρουσίασης**  
Η κλάση `Presentation` είναι το αντικείμενο υψηλότερου επιπέδου του Aspose.Slides που αντιπροσωπεύει ένα αρχείο PowerPoint στη μνήμη.  
```java
Presentation pres = new Presentation();
```
*Δημιουργεί το αντικείμενο `Presentation` που θα περιέχει όλες τις διαφάνειες.*

**Βήμα 2: Προσθήκη Γραφήματος Πίτας**  
`ChartType.Pie` λέει στο Aspose να αποδώσει ένα γράφημα πίτας.  
```java
IChart chart = pres.getSlides().get_Item(0).getShapes().addChart(
    ChartType.Pie,
    50,
    50,
    400,
    500
);
```
*Τοποθετεί ένα γράφημα πίτας στις συντεταγμένες (50, 50) με πλάτος 400 και ύψος 500.*

**Βήμα 3: Αποδέσμευση Πόρων**  
Η κλήση του `dispose()` απελευθερώνει τους εγγενείς πόρους και αποτρέπει διαρροές μνήμης.  
```java
if (pres != null) pres.dispose();
```
*Απελευθερώνει τους εγγενείς πόρους· πάντα καλέστε `dispose()` όταν τελειώσετε.*

### Δυνατότητα 2: Πρόσβαση στο Βιβλίο Εργασίας Δεδομένων Γραφήματος και στα Φύλλα Εργασίας
#### Επισκόπηση
Μάθετε πώς να φτάσετε στο υποκείμενο βιβλίο εργασίας που αποθηκεύει τα δεδομένα του γραφήματος και να επαναλάβετε τα φύλλα εργασίας του.

#### Πώς να αποκτήσετε πρόσβαση στο βιβλίο εργασίας δεδομένων γραφήματος;
Ανακτήστε το `IChartDataWorkbook` από το γράφημα, στη συνέχεια επαναλάβετε τη συλλογή `Worksheets`. Αυτό το βιβλίο εργασίας μιμείται ένα αρχείο Excel, επιτρέποντάς σας να διαβάζετε, τροποποιείτε ή προσθέτετε σειρές δεδομένων προγραμματιστικά, κάτι που το γράφημα θα αντικατοπτρίζει άμεσα όταν ανανεώνεται κατά την εκτέλεση χωρίς επανεκκίνηση.

#### Βήμα‑βήμα

**Βήμα 1: (Επανάχρηση) Αρχικοποίηση Νέου Αντικειμένου Παρουσίασης**  
*Ίδιο με τη Δυνατότητα 1, Βήμα 1.*

**Βήμα 2: (Επανάχρηση) Προσθήκη Γραφήματος Πίτας**  
*Ίδιο με τη Δυνατότητα 1, Βήμα 2.*

**Βήμα 3: Λήψη του Βιβλίου Εργασίας Δεδομένων Γραφήματος**  
`IChartDataWorkbook` είναι η διεπαφή που παρέχει πρόσβαση ανάγνωσης/εγγραφής στο εσωτερικό βιβλίο εργασίας του γραφήματος, παρόμοιο με το Excel.  
```java
IChartDataWorkbook workbook = chart.getChartData().getChartDataWorkbook();
```
*Ανακτά το `IChartDataWorkbook` που συνδέεται με το γράφημα.*

**Βήμα 4: Επανάληψη μέσω Φύλλων Εργασίας**  
Τα αντικείμενα `Worksheet` αντιπροσωπεύουν μεμονωμένα φύλλα μέσα στο βιβλίο εργασίας.  
```java
for (int i = 0; i < workbook.getWorksheets().size(); i++) {
    System.out.println(workbook.getWorksheets().get_Item(i).getName());
}
```
*Εμφανίζει το όνομα κάθε φύλλου εργασίας, επιτρέποντάς σας να επαληθεύσετε τη δομή των δεδομένων.*

**Βήμα 5: Αποδέσμευση Πόρων**
