## Руководство по внедрению

Давайте разберем процесс настройки и использования правил резервного копирования шрифтов в Aspose.Slides для .NET.

### Создание шрифта FallBackRulesCollection

Основная функция — создание коллекции, определяющей, как ваше приложение должно обрабатывать шрифты, недоступные в системе. 

#### Обзор

Правила резервного копирования шрифтов необходимы, когда вы хотите обеспечить правильное отображение определенных шрифтов, особенно для нестандартных символов или сценариев.

##### Шаг 1: Инициализация FontFallBackRulesCollection

Начните с инициализации нового `IFontFallBackRulesCollection` объект:

```csharp
using (Presentation presentation = new Presentation())
{
    IFontFallBackRulesCollection userRulesList = new FontFallBackRulesCollection();
}
```

#### Добавление резервных правил

Чтобы добавить правила резервного копирования шрифтов, используйте `Add()` Метод. Позволяет указать диапазоны Unicode и соответствующие шрифты.

##### Шаг 2: Определите пользовательские резервные правила

1. **Сопоставление диапазона Unicode U+0B80-U+0BFF со шрифтом «Vijaya»**
   
   Это правило гарантирует, что для символов в этом диапазоне Unicode по умолчанию будет использоваться шрифт «Vijaya», если он доступен:
   
   ```csharp
   userRulesList.Add(new FontFallBackRule(0x0B80, 0x0BFF, "Vijaya"));
   ```

2. **Сопоставление диапазона Unicode U+3040-U+309F с "MS Mincho, MS Gothic"**
   
   Это правило охватывает символы в указанном диапазоне и сопоставляет их либо с «MS Mincho», либо с «MS Gothic»:
   
   ```csharp
   userRulesList.Add(new FontFallBackRule(0x3040, 0x309F, "MS Mincho, MS Gothic"));
   ```

#### Назначение резервных правил для презентации

После настройки правил назначьте их менеджеру шрифтов презентации:

```csharp
presentation.FontsManager.FontFallBackRulesCollection = userRulesList;
```

## Раздел часто задаваемых вопросов

1. **Что такое правило резервного шрифта?**
   
   Правило резервного шрифта определяет альтернативные шрифты, которые следует использовать, когда основной шрифт недоступен для определенных символов.

2. **Как протестировать правила резервного копирования шрифтов?**
   
   Создайте образцы документов, содержащих определенные диапазоны Unicode, и проверьте их отображение на разных платформах.

3. **Может ли Aspose.Slides обрабатывать все диапазоны Unicode?**
   
   Да, но убедитесь, что вы сопоставили каждый требуемый диапазон с соответствующими шрифтами.

4. **Что делать, если шрифт недоступен?**
   
   Убедитесь, что резервные правила настроены правильно, или включите необходимые шрифты в свой дистрибутив.

5. **Существует ли ограничение на количество резервных правил?**
   
   Строгих ограничений нет, но чрезмерные правила могут повлиять на производительность и использование памяти.
