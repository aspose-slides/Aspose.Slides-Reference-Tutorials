## Руководство по внедрению
### Обзор обработки токенов прерывания
Маркеры прерывания позволяют вашему приложению приостанавливать или останавливать определенные задачи изящно. Это особенно полезно при обработке больших презентаций, когда пользователю может потребоваться отменить операцию до ее завершения.

### Пошаговая реализация
#### 1. Инициализация источника токена прерывания
Сначала создайте `InterruptionTokenSource` для мониторинга и устранения перерывов:
```java
import com.aspose.slides.InterruptionTokenSource;

final InterruptionTokenSource tokenSource = new InterruptionTokenSource();
```
#### 2. Создание выполняемой задачи
Определите задачу, которая загружает и обрабатывает презентацию:
```java
Runnable task = () -> {
    // Создайте параметры загрузки с токеном прерывания.
    LoadOptions options = new LoadOptions();
    options.setInterruptionToken(tokenSource.getToken());

    // Загрузите презентацию, используя указанный путь и параметры.
    Presentation presentation = new Presentation("YOUR_DOCUMENT_DIRECTORY/pres.pptx", options);
    try {
        // Сохраните презентацию в другом формате.
        presentation.save("YOUR_OUTPUT_DIRECTORY/pres.ppt", SaveFormat.Ppt);
    } finally {
        if (presentation != null) presentation.dispose();
    }
};
```
#### 3. Выполнение и прерывание задачи
Выполнить задачу в отдельном потоке и смоделировать прерывание после некоторой задержки:
```java
Thread thread = new Thread(task); // Запустите задачу в отдельном потоке.
thread.start();

Thread.sleep(10000); // Смоделируйте выполнение какой-либо работы до прерывания.

// Вызвать прерывание, влияющее на текущую обработку.
tokenSource.interrupt();
```
### Объяснение ключевых компонентов
- **Источник маркера прерывания:** Управляет состоянием прерываний и взаимодействует с выполняющейся задачей.
- **LoadOptions.setInterruptionToken():** Связывает токен прерывания с операциями загрузки презентации.
- **Представление.dispose():** Обеспечивает надлежащее высвобождение ресурсов даже в случае прерывания.

### Советы по устранению неполадок
К распространенным проблемам относятся:
- Неправильный путь к презентациям: убедитесь, что пути верны.
- Неправильно настроенные потоки: проверьте управление потоками и обработку исключений в вашем приложении.
