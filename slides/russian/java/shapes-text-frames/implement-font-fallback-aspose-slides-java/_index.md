## Руководство по внедрению
### Создание правил резервного копирования шрифтов
#### Обзор
Настройка правил резервного копирования шрифтов гарантирует, что ваши презентации будут отображать текст правильно, даже если определенные шрифты недоступны в системе пользователя. Это имеет решающее значение при работе с нелатинскими шрифтами или специализированными символами.
#### Добавление определенных правил резервного копирования шрифтов
Создать экземпляр `FontFallBackRulesCollection` и добавьте пользовательские правила:
**Шаг 1: Инициализация коллекции**
```java
import com.aspose.slides.FontFallBackRulesCollection;
FontFallBackRulesCollection userRulesList = new FontFallBackRulesCollection();
```
**Шаг 2: Добавьте правила для диапазонов Unicode**
Сопоставьте определенные диапазоны Unicode с желаемыми шрифтами:
- **Правило 1**: Сопоставить тамильский алфавит (диапазон Unicode от 0x0B80 до 0x0BFF) со шрифтом «Vijaya».
```java
userRulesList.add(new FontFallBackRule(0x0B80, 0x0BFF, "Vijaya"));
```
- **Правило 2**: Преобразовать хирагану/катакану (диапазон Unicode от 0x3040 до 0x309F) в «MS Mincho» или «MS Gothic».
```java
userRulesList.add(new FontFallBackRule(0x3040, 0x309F, "MS Mincho, MS Gothic"));
```
**Шаг 3: Примените правила**
Установите следующие правила в менеджере шрифтов вашей презентации:
```java
presentation.getFontsManager().setFontFallBackRulesCollection(userRulesList);
```
### Советы по устранению неполадок
- **Отсутствующие шрифты**Убедитесь, что все указанные резервные шрифты установлены в системе.
- **Несоответствие Unicode**: Убедитесь, что диапазоны Unicode соответствуют требованиям вашего скрипта.
