# แปลง PPTX เป็น HTML พร้อมรักษาฟอนต์ไว้โดยใช้ Aspose.Slides สำหรับ Python

## การแนะนำ

การแปลงงานนำเสนอ PowerPoint (PPTX) เป็นรูปแบบ HTML โดยยังคงรักษาแบบอักษรดั้งเดิมไว้อาจเป็นเรื่องท้าทาย โดยเฉพาะอย่างยิ่งหากคุณต้องการหลีกเลี่ยงการฝังแบบอักษรเริ่มต้นบางตัว ด้วย "Aspose.Slides สำหรับ Python" งานนี้จะกลายเป็นเรื่องง่ายขึ้น บทช่วยสอนนี้จะแนะนำคุณเกี่ยวกับการแปลงไฟล์ PPTX เป็น HTML ด้วยแบบอักษรที่เก็บรักษาไว้โดยใช้ Aspose.Slides ใน Python

**สิ่งที่คุณจะได้เรียนรู้:**
- วิธีการติดตั้งและตั้งค่า Aspose.Slides สำหรับ Python
- การแปลงงานนำเสนอ PowerPoint (PPTX) เป็น HTML โดยยังคงรักษาแบบอักษรไว้
- ไม่รวมแบบอักษรเริ่มต้นเฉพาะจากการฝัง
- การเพิ่มประสิทธิภาพการทำงานในระหว่างกระบวนการแปลง

มาทบทวนข้อกำหนดเบื้องต้นกันก่อนเริ่มต้น!

## คู่มือการใช้งาน

ในส่วนนี้เราจะแบ่งกระบวนการแปลงออกเป็นขั้นตอนที่สามารถจัดการได้

### การแปลง PPTX เป็น HTML โดยรักษาฟอนต์ดั้งเดิม

#### ภาพรวม:
คุณสมบัติหลักของการใช้งานนี้คือการแปลงงานนำเสนอ PowerPoint โดยยังคงรักษาแบบอักษรดั้งเดิมไว้และไม่รวมแบบอักษรเริ่มต้นบางแบบเข้าไป ซึ่งอาจมีประโยชน์โดยเฉพาะในการรักษาความสอดคล้องของแบรนด์ในงานนำเสนอบนเว็บ

#### การดำเนินการทีละขั้นตอน:

**1. กำหนดเส้นทางอินพุตและเอาต์พุต**

ตั้งค่าไดเร็กทอรีที่ไฟล์ PPTX อินพุตของคุณอยู่และที่คุณต้องการบันทึกไฟล์ HTML เอาท์พุต

```python
data_dir = "YOUR_DOCUMENT_DIRECTORY/"
out_dir = "YOUR_OUTPUT_DIRECTORY/"
```

**2. เปิดไฟล์การนำเสนอ**

ใช้ Aspose.Slides' `Presentation` คลาสสำหรับโหลดไฟล์ PPTX ของคุณ:

```python
with slides.Presentation(data_dir + "welcome-to-powerpoint.pptx") as pres:
    # รหัสการแปลงของคุณจะอยู่ที่นี่
```

ตัวจัดการบริบทนี้จะช่วยให้แน่ใจว่าทรัพยากรจะได้รับการปล่อยอย่างถูกต้องหลังการดำเนินการ

**3. สร้างตัวควบคุมการฝังแบบอักษรที่กำหนดเอง**

ไม่รวมแบบอักษรบางตัวจากการฝังโดยใช้ `EmbedAllFontsHtmlController`-

```python
font_name_exclude_list = ["Calibri", "Arial"]
embed_fonts_controller = slides.export.EmbedAllFontsHtmlController(font_name_exclude_list)
```

ที่นี่ "Calibri" และ "Arial" จะถูกแยกออกจากการฝังในผลลัพธ์ HTML

**4. กำหนดค่าตัวเลือกการส่งออก HTML**

ตั้งค่า `HtmlOptions` ในการใช้ตัวจัดรูปแบบแบบอักษรที่กำหนดเองกับตัวควบคุมของคุณ:

```python
html_options_embed = slides.export.HtmlOptions()
html_options_embed.html_formatter = slides.export.HtmlFormatter.create_custom_formatter(embed_fonts_controller)
```

ขั้นตอนนี้จะช่วยให้แน่ใจว่ามีเพียงแบบอักษรที่จำเป็นเท่านั้นที่ฝังอยู่ในผลลัพธ์สุดท้าย

**5. บันทึกการนำเสนอเป็น HTML**

สุดท้าย ให้บันทึกการนำเสนอลงในไฟล์ HTML พร้อมตัวเลือกที่คุณระบุ:

```python
pres.save(out_dir + "convert_to_html_with_preserving_original_fonts_out.html", 
          slides.export.SaveFormat.HTML, html_options_embed)
```

### เคล็ดลับการแก้ไขปัญหา:
- ตรวจสอบให้แน่ใจว่าเส้นทางได้รับการตั้งค่าอย่างถูกต้องและสามารถเข้าถึงได้
- ตรวจสอบไฟล์แบบอักษรที่หายไปบนระบบซึ่งอาจส่งผลต่อการแปลง

## การประยุกต์ใช้งานจริง

ต่อไปนี้คือสถานการณ์จริงบางสถานการณ์ที่ฟีเจอร์นี้อาจเป็นประโยชน์อย่างยิ่ง:

1. **เว็บพอร์ทัล**:แปลงงานนำเสนอเป็น HTML เพื่อบูรณาการเข้ากับแอปพลิเคชันเว็บได้อย่างราบรื่นโดยไม่สูญเสียแบบอักษรของแบรนด์
2. **ระบบจัดการเอกสาร**:ฝังการนำเสนอลงในพอร์ทัลภายในพร้อมรักษาความถูกต้องของเอกสาร
3. **แพลตฟอร์มการเรียนรู้ทางอิเล็กทรอนิกส์**:ใช้ไฟล์ HTML ที่แปลงแล้วเป็นส่วนหนึ่งของหลักสูตรออนไลน์ โดยคงรูปลักษณ์และความรู้สึกที่สอดคล้องกัน

เพื่อให้แน่ใจว่าได้ประสิทธิภาพสูงสุดระหว่างการแปลง:

- **ใช้เวอร์ชันไลบรารี่ล่าสุด**:ใช้ Aspose.Slides เวอร์ชันล่าสุดอยู่เสมอเพื่อปรับปรุงคุณสมบัติและแก้ไขข้อบกพร่อง
